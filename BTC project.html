Now, I will attempt a very crude method of predicting whether a given observation will increase or decrease. First, I will assign each known data-point the label 'increasing' or 'decreasing'. This, for days where data on how prices changed within a single day is avaliable, will be determined by checking if the price increased or decreased over the day. For those where no such data is avaliable, it will be determined by checking if the price increases the following day. Then, I will try a couple of different models, including logistic regression and random forests to try to sort out the most important factors in determining whether a coin will increase in value the following day.

I am unsure of whether this method will prove useful--after all, it does not take into account anything but the data of a single observation when making a prediction. It will not be able to account for long-term trends or seasonal cycles. It cannot explain market forces. However, I believe it is plausibly helpful as an exploration of the important factors in predicting price

As a first step, we will label the parameters. Then, prune away the parameters we do not want to use.

First, we will use a random-forest model to predict prices based off of an observation directly. Then, we will try both a logistic regression model and a random forest model to predict whether an observation is increasing.

We will use K-fold validation-- the dataset will be randomly broken up 10 times into training data and test data. This will help us prevent overfitting.

#FIRST- DIRECT PRICE PREDICTION with 10-fold cross validation
from sklearn.model_selection import KFold
from sklearn.ensemble import RandomForestRegressor
from sklearn.tree import export_graphviz
#BTC
x = btc_data.iloc[:,[2,5,10,11,16]]
#x = btc_data.iloc[:,2:21]
y = btc_data['btc_market_price']

kf = KFold(10, shuffle=True)
scores = []
model = RandomForestRegressor(10)

for train, test in kf.split(btc_data):
    xtrain = x.iloc[train]
    xtest = x.iloc[test]
    ytrain = y.iloc[train]
    ytest = y.iloc[test]
    model.fit(xtrain,ytrain)
    scores.append(model.score(xtest, ytest))
print("btc model error: "+ str(np.mean(scores)))


#ETH
x = eth_data.iloc[:,[4,7,8,16]]
y = eth_data['eth_etherprice']
scores = []
for train, test in kf.split(eth_data):
    xtrain = x.iloc[train]
    xtest = x.iloc[test]
    ytrain = y.iloc[train]
    ytest = y.iloc[test]
    model.fit(xtrain,ytrain)
    scores.append(model.score(xtest, ytest))
print("eth model error: "+ str(np.mean(scores)))
